# Bigmart Sales Prediction

A machine learning regression project predicting retail sales revenue using the Bigmart Sales dataset (8,523 records). This project documents statistical feature selection, handling missing data through pattern investigation rather than blind imputation, and comparing three regression models, including a case where added model complexity genuinely improved results.

## Project Overview

Predicting Item_Outlet_Sales, the revenue generated by a specific product at a specific outlet, using features about the item (price, type) and the outlet (type, size, location). Unlike a previous e-commerce project, where most features showed weak predictive signal, this dataset contained genuinely strong relationships, Item_MRP correlated at 0.57 with sales, and Outlet_Type showed a nearly tenfold difference in average sales between the best and worst performing outlet formats.

## Methodology

Features were selected using Pearson correlation for numeric features and ANOVA for categorical features, the same evidence-based approach used in prior work. Missing values in Outlet_Size were investigated for patterns before being filled: Grocery Store outlets were unanimously one size, allowing a confident fill, while Supermarket Type1 outlets were split across multiple sizes, too ambiguous to guess, so those were labeled as their own category instead. A redundancy check confirmed Outlet_Identifier added nothing beyond the simpler Outlet_Type, so the more granular column was dropped in favor of the broader one.

Key techniques used:
- Pearson correlation and ANOVA for feature selection
- Pattern investigation for missing data, rather than uniform imputation
- Target encoding for the high-cardinality Item_Type feature, calculated only from training data
- Comparison of Linear Regression, Random Forest, and Gradient Boosting
- Feature importance analysis to validate the model's behavior against the statistical findings

## Results

| Model | R² | MAE | RMSE |
|---|---|---|---|
| Linear Regression | 0.580 | 791.44 | 1068.61 |
| Random Forest | 0.611 | 718.02 | 1028.20 |
| Gradient Boosting | 0.603 | 721.99 | 1039.22 |

Random Forest was selected as the final model. Feature importance showed Item_MRP and Outlet_Type together accounted for about 98 percent of the model's decisions, with the remaining features, despite being statistically significant, contributing very little once the two dominant features were already included.

## Key Takeaway

Unlike a previous project where multiple models converged to the same low ceiling regardless of approach, this dataset showed real benefit from added model complexity, both Random Forest and Gradient Boosting meaningfully outperformed the linear baseline. Recognizing the difference between a dataset with weak signal and one with strong, partially non-linear signal is itself a useful skill, since it determines how much modeling effort is actually worth investing.
